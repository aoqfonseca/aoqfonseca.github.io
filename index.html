Como fazer sistemas prontos para produção

Perfomance.

O tempo de resposta e a capacidade do seu sistema pode ser a diferença entre o sucesso ou fracasso. Existem vários casos que apoiam essa afirmação

Resiliência

Sistema devem ser capazes de se recuperar de falhas e suportar indisponibilidades em suas dependencias, sem afetar o serviço entregue ao usuário final.

Monitoração e logs

Não é possível ter sucesso sem medir ou monitorar. Não tem como melhorar sem ter números e dados para analisar. Monitorações e logs são a base de tudo.

  • Sistema prontos

    o que são sistemas prontos para produção

    Ter testes automatizado, qualidade de código, design, etc, não são as unicas coisas para termos sistema realmente prontos para produção. Nessa parte vamos ver o que é necessário para um sistema ser considerado PRONTO.

  • Perfomance

    Aspectos de sistemas prontos

    Um dos principais pontos de sistemas prontos para produção é a questão de perfomance. Sem isso, não é possível pensar nos demais passos. Sistema precisam estar preparados para as cargas de usuários e serem capazes de responder rápido.

  • Resiliencia

    Porque não devemos cair por qualquer problema

    Sistema não são ilhas e nem vivem isolados. Nos modelos atuais, qualquer sistema dependem de um enormidade de outros. Com isso lidar com indisponibilidades, tempos altos, erros externos é vital para termos sistemas prontos para produção.

  • Monitoração e Logs

    Se não medir não tem como saber se está no caminho certo

    Não tem como termos um sistema pronto para produção sem ter monitoração e logs que conseguimos extrair informação. Muita gente não sabe a real importancia de um log até o dia que o sietma dá erro. Monitorar é a chave também. Vamos ver tudo isso.
